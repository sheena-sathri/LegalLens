## 6 MODULES:

### 1. 🚪 AI Legal Front Door (Intake & Triage) — THIS IS THE HOME PAGE
- A clean chat-style intake interface where a user types their legal request in natural language (e.g., "I need an NDA reviewed for a new vendor", "What's our policy on data retention?", "I need to get a contract approved urgently")
- When the user submits a request, AI auto-classifies it into a matter type: Contract Review, NDA Request, Policy Question, Compliance Check, IP Review, Employment Matter, Vendor Agreement, Litigation Hold, etc.
- Shows the classification result with confidence score, urgency level (Low/Medium/High/Urgent), and recommended routing (which team/person/workflow it should go to)
- Below the chat, show a "Recent Requests" feed showing the last 5-10 requests with their status (New, Triaged, In Progress, Resolved)
- Also show quick-action buttons: "Request NDA", "Policy Question", "Contract Review", "Report an Issue" — these are self-service shortcuts
- Mock Slack integration panel on the side: shows a simulated Slack conversation where someone types "#legal I need an NDA for Acme Corp" and the bot responds with "Got it! I've created matter #M-2024-0142. Classified as: NDA Request (High confidence). Assigned to: Sarah, Contracts Team. ETA: 2 business days. Track status here: [link]"
- Mock Email integration panel: shows a parsed incoming email from "john@company.com" with subject "Need contract reviewed ASAP" that gets auto-parsed into a matter with extracted details (sender, urgency, matter type, attachments)

### 2. 📋 Matter Management
- A Kanban board with columns: New | In Review | Pending Approval | Approved | Closed
- Each matter card shows: Matter ID (#M-2024-0142), Title, Type (NDA, MSA, etc.), Requester name, Assigned attorney, Priority badge (color-coded), Due date, Number of linked documents
- Clicking a matter opens a detail view with:
  - Matter summary (AI-generated one-liner)
  - Timeline/activity feed (chronological log of all actions: "Created via Slack intake", "Auto-classified as NDA Request", "Assigned to Sarah Chen", "Document uploaded: vendor_nda_v2.pdf", "AI clause extraction complete — 2 HIGH risk flags", "Comment by Sarah: Need to renegotiate liability cap")
  - Linked documents section — shows uploaded docs with their classification and risk summary
  - This is where the Document Intelligence features live: upload a doc to the matter, auto-classify it, extract clauses, show risk scores — all tied to this specific matter
  - Tasks/checklist (e.g., "Review termination clause", "Get VP approval", "Send to counterparty")
  - Notes section for internal lawyer notes
- Use mock/demo data to populate 8-10 sample matters across different statuses so the board looks real and alive
- Include a table/list view toggle as alternative to Kanban

### 3. 💬 AI Legal Assistant (Chatbot)
- Full chat interface — looks like a messaging app
- The chatbot can:

  - Answer document questions: "Summarize the key risks in matter #M-2024-0142" → uses RAG to pull from uploaded docs
  - Help with intake: "I need to get a contract reviewed" → walks the user through creating a new matter
  - Provide self-service links: "How do I request an NDA?" → provides the workflow link
- Show a "Sources" section below each AI response with citations (which document, which section, confidence score)
- Chat history in the sidebar showing past conversations
- At the bottom, show a disclaimer: "AI responses are for guidance only. Always consult with your legal team for final decisions."
- Also show a "Chatbot on Slack" preview panel: simulated Slack DM with the legal bot showing the same kind of Q&A
- Also show a "Chatbot on Email" preview: shows how the bot responds to email queries with structured answers

### 6. 🔒 Privacy & LLM Safety — THIS IS THE KEY DIFFERENTIATOR
This page explains how the platform keeps legal data safe. It should be visual, clear, and written for non-technical lawyers who are nervous about AI.

Section 1: "How Your Data Flows"
- Visual architecture diagram (use Streamlit columns and styled containers to simulate this):
  - Show: User uploads document → Document stored in YOUR infrastructure → Text extracted locally → Chunks created → Sent to LLM for analysis → Response returned → Audit logged
  - Emphasize with green checkmarks: "✅ Documents stored on your servers", "✅ No data retained by LLM provider", "✅ Every AI interaction logged"
  - Red X marks: "❌ No training on your data", "❌ No third-party access", "❌ No data leaves your network (in self-hosted mode)"

Section 2: "Deployment Modes"
- Two cards side by side:
  - 🔒 SELF-HOSTED / AIR-GAPPED: "LLM runs locally (Llama 3, Mistral). All data stays on-premises. Zero external API calls. For organizations where data can never leave the perimeter. Suitable for: Defense, Healthcare, Financial Services, and any org with strict data residency requirements."
  - 🔗 HYBRID: "Connects to external LLM APIs (Claude, GPT) with zero-retention agreements. Integrates with CLM/DMS tools (Ironclad, iManage, DocuSign) via secure API connectors. Full control over which data goes where. Suitable for: Tech companies, enterprises with vetted vendor programs."

Section 3: "What the AI Sees"
- A transparency panel showing:
  - "When you upload a contract, here's exactly what happens:"
  - Step 1: Document text extracted (locally, on your server)
  - Step 2: Text split into sections (no PII sent to LLM unless necessary)
  - Step 3: PII Detection runs first — names, SSNs, account numbers flagged and optionally redacted before LLM processing
  - Step 4: Relevant sections sent to LLM with task-specific prompt
  - Step 5: LLM returns structured analysis (classification, clauses, risk scores)
  - Step 6: Full audit log recorded (input hash, output, timestamp, model used)
  - "The LLM never sees your full document — only the relevant sections needed for the specific task."

Section 4: "Integrations & Security"
- Grid of integration cards showing supported connectors:
  - Slack (intake, chatbot), Email (intake, notifications), Ironclad (CLM sync), iManage (document management), DocuSign (e-signatures), Salesforce (matter tracking), Jira (task management), SSO (Okta, Azure AD)
  - Each card shows a toggle: Connected / Not Connected
  - A note: "All integrations use encrypted API connections. Data flows are configurable — you control exactly what syncs where."

Section 5: "Audit & Compliance"
- Show the existing governance/audit trail data
- Every AI query logged with: timestamp, user, action, document (hashed), model used, confidence score, latency
- Export to CSV button
- "Designed for SOC 2, ISO 27001, GDPR, and CCPA compliance requirements"

Section 6: "Model Selection Guide"
- A comparison table:
  | Consideration | Self-Hosted (Llama/Mistral) | API (Claude/GPT) |
  | Data residency | ✅ All on-prem | ⚠️ Sent to provider (with zero-retention) |
  | Performance | Good | Best |
  | Cost | Hardware + maintenance | Per-token API fees |
  | Setup complexity | Higher | Lower |
  | Best for | Max security, regulated industries | Speed, quality, lower ops burden |
